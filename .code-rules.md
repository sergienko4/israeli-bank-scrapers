# Code Review Standards — israeli-bank-scrapers

> **Quality caps:** Function size, complexity, parameter count, and file-size
> limits are documented in [CLEAN_CODE.md](./CLEAN_CODE.md) — the single source
> of truth. The pre-commit `lint:guideline-coverage` gate asserts
> `eslint.config.mjs` actually enforces those caps for every Pipeline cluster.

## Architecture Rules (ABSOLUTE)

- NO CSS selectors (class, ID, attribute) in click/fill/navigate/wait interaction code
- Use visible text selectors: getByText(), getByRole(), getByPlaceholder()
- Structural CSS selectors ONLY allowed in parsing/extraction code (pageEvalAll, date picker grids, frame detection)

## TypeScript Strict Mode

- No `any` type — use specific types or `JsonValue`
- No unused variables or imports
- Explicit return types on all exported functions
- No non-null assertions (`!`) — use explicit null checks

## Function Quality

See [CLEAN_CODE.md](./CLEAN_CODE.md) for canonical caps (file size, per-function
LoC, complexity, parameter count, etc.). Do NOT restate the numbers here — the
caps live in exactly one place to prevent the doc/config drift that caused
CR cycle-1 #7 on PR #278.

- Extract helpers when a function exceeds the cluster cap
- Use options object beyond 3 parameters
- Split complex logic; use lookup tables over if/else ladders
- JSDoc on every exported function

## Async Patterns (CRITICAL)

- Check return values from boolean async functions — never ignore them
- No `await` in loops — use `runSerial` or `Promise.all`
- Stop iterating after first successful action (short-circuit)
- Wait for DOM elements to be visible before interacting

## Security

- Never embed unsanitized strings in XPath — use `toXpathLiteral()`
- Never use `eval()`, `Function()`, or `child_process.exec()` with user input
- Never hardcode credentials or API keys in source
- Validate parsed JSON before casting to typed interfaces

## Error Handling

- Never swallow errors silently — log or rethrow
- Use `ScraperError` class, not bare `throw new Error()`
- Check for element existence before interaction (count > 0)
- Propagate failures — don't return `success: true` when operations fail
